GTA Hiring Process

Workflow Redesign for Administrative Efficiency

Timeline

September 2025 - December 2025

Team

Solo Project

Role

UX Researcher & Designer

Skills

Figma, Power Automate, Excel, User Research

LONG STORY SHORT

Administrative staff at Georgia Tech's School of Interactive Computing spend 8-12 hours per hiring cycle manually managing GTA applications through error-prone spreadsheets. Faculty submit inconsistent data via email, leading to preventable delays that impact student payment and academic appointments. This project redesigned the entire workflow using Microsoft's existing tools (Forms, Power Automate, and Excel) to reduce errors by 85% and save 2-3 hours per cycle.

THE CHALLENGE

Admin coordinators manually tracked 20-30 GTA hires per semester using shared spreadsheets vulnerable to data loss, formatting errors, and version conflicts. Faculty submitted inconsistent information via email, requiring hours of data cleaning and follow-up.

THE SOLUTION

An automated workflow using Microsoft Forms for validated data entry, Power Automate for notifications and DocuSign generation, and a protected Excel master sheet as the single source of truth.

UNDERSTANDING THE PROBLEM

I conducted semi-structured interviews with 2 admin coordinators and multiple faculty members, combined with contextual inquiry observing the actual hiring process in real-time.

KEY FINDINGS

  • Heavy reliance on manual spreadsheet work consuming 8-12 hours per cycle
  • Faculty submit data in inconsistent formats (phone numbers, GTIDs, emails)
  • Broad edit access creates risk of accidental deletions
  • Teams chat buried critical status updates in long threads
  • New hires/transfers require intensive coordination with ASC

RESEARCH METHODS

  • Semi-structured interviews with admin staff and faculty
  • Contextual inquiry observing live hiring cycles
  • Artifact analysis of spreadsheets, Teams chats, email threads
  • Comparative analysis of similar processes (GRA hiring)
  • Affinity mapping to identify themes

RESEARCH THEMES

Analysis revealed six systemic issues preventing administrative staff from working efficiently:

Process Complexity & Manual Work

Row-by-row data entry and non-standard formats consume hours of manual work

MSFT Dependence & Communication

Teams chat buries status updates in threads; color codes inconsistently applied

System Vulnerabilities

Shared spreadsheets have broad edit access, risking deletions and conflicting changes

Ineffective Error Correction

Errors caught late require repeated faculty follow-up, delaying hiring

Limited Automation

Available Microsoft tools (Forms, Power Automate) not being used

Organization & Coordination

One admin carries workload; constant vigilance required to avoid mistakes

DESIGN OPPORTUNITY

How might we redesign the GTA hiring process so administrative staff could minimize errors while maintaining scalability?

USER NEEDS

  • Validated data entry to prevent formatting errors
  • Automated notifications instead of manual follow-ups
  • Single source of truth protected from accidental edits
  • Real-time status tracking visible to all stakeholders
  • Automatic DocuSign generation from clean data

DESIGN CONSTRAINTS

  • Must stay within Georgia Tech's Microsoft ecosystem
  • Respect data governance and policy requirements
  • No additional software costs or purchases
  • Admin must retain control and override capability
  • Faculty must understand the changed process

SOLUTION DESIGN

I designed an integrated system using three Microsoft tools already available at Georgia Tech:

Microsoft Forms

Validated data entry with required fields, format checks (GTID, phone, email), conditional logic, and clear helper text

Power Automate

Workflow automation sending form links, writing to master sheet, monitoring status changes, triggering DocuSign, logging events

Excel Master Sheet

Protected single source of truth with structured schema, role-based access, duplicate checks, status columns, change history

PROCESS COMPARISON

Before (Old Process)

  • Admin emails spreadsheet to faculty
  • Faculty fills out student info (risk: overwriting/deleting, missing/incorrect format)
  • Faculty returns spreadsheet (risk: missing info, deadlines)
  • Admin reviews/corrects data (pain: lots of manual reformatting, phone/email errors)
  • Admin sends offer via DocuSign
  • Student accepts/declines (admin must update status accordingly)
  • For accepted: admin initiates hire with ASC (new hires/transfers require the most attention)
  • For extensions/transfers: admin updates records, marks extension status
  • All along, communication and troubleshooting occur in Teams chat
  • In case of critical data problems, admin must pause editing and contact IT to recover data

After (New Process)

  • Admin creates GTA master Copilot-integrated Excel sheet (admin-only access)
  • Admin creates validated Outlook Form with auto-validation, support fields, dropdowns, and format instructions
  • Copilot drafts emails, schedules deadline, reminder automation
  • Faculty entries on Form (forms prevent invalid data)
  • Submissions entered into admin's master sheet
  • Copilot monitors incoming applications (should be able to flag data)
  • Copilot writes status summaries (including corrections pending, offer ready list)
  • Power Automate (MSFT) triggers DocuSign offer letter generation
  • Copilot and Power Automate send conditional DocuSign status update
  • Admin reviews and approves
  • All offers signed? Yes → Admin notifies ASC/HR for onboarding / No → [loop back]

WHAT WORKED

  • Contextual inquiry revealed the gap between stated and actual process
  • Affinity mapping uncovered systemic issues, not individual mistakes
  • Staying within Microsoft ecosystem ensured feasibility
  • Storyboards helped stakeholders visualize the transformation

KEY TAKEAWAYS

  • Back-office systems deserve the same UX rigor as consumer products
  • Admin role shifts from "error fixer" to "exception manager"
  • Automation works best when users retain control and visibility
  • Documentation and training are essential for adoption
